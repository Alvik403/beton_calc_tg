# Формулы, используемые в проекте

## 1. Калькулятор максимума объёма бетона (`app/calculator.py`)

### 1.1. Лимит по одному материалу (сколько м³ можно сделать)

Для каждого материала в рецепте: сколько кубометров бетона хватает при текущем остатке:

```
лимит_по_материалу [м³] = остаток_материала [кг] / расход_на_м³ [кг/м³]
```

где:
- **остаток_материала** — сколько кг материала есть на складе (из Excel, столбец «Сальдо на конец периода»);
- **расход_на_м³** — сколько кг этого материала нужно на 1 м³ бетона (из рецепта в `config/recipes.yaml`).

Если `расход_на_м³ ≤ 0`, материал в расчёте не участвует.

### 1.2. Максимальный объём по рецепту

Ограничивающим фактором служит материал, которого хватает на наименьший объём:

```
максимум_м³ = min(лимит_1, лимит_2, …, лимит_N)
```

То есть для каждого материала считается `остаток / расход_на_м³`, затем берётся **минимум** по всем материалам рецепта.

### 1.3. Требуемое количество материалов для этого объёма

Для выбранного максимума м³ считается, сколько каждого материала нужно:

```
нужно_материала [кг] = максимум_м³ × расход_на_м³ [кг/м³]
```

По сути:

```
нужно_материала = максимум_м³ × recipe.materials[материал]
```

## 2. Стоимость (`app/web.py`, лист «Итог»)

Цены в конфиге заданы **за 1 м³** (руб/м³). Итоговая стоимость для посчитанного объёма:

```
стоимость = максимум_м³ × цена_за_м³
```

Используются четыре варианта цен (из `config/prices.yaml`):

| Поле в отчёте | Формула |
|---------------|--------|
| Стоимость без доставки без НДС | `максимум_м³ × no_delivery_no_vat` |
| Стоимость без доставки с НДС 22% | `максимум_м³ × no_delivery_vat_22` |
| Стоимость самовывоз без НДС | `максимум_м³ × pickup_no_vat` |
| Стоимость самовывоз с НДС 22% | `максимум_м³ × pickup_vat_22` |

НДС в формулах не пересчитывается — в конфиге уже задана цена с НДС или без.

## 3. Парсинг чисел из Excel (`app/excel_parser.py`)

### 3.1. Преобразование значения ячейки в число

- Если значение уже число (int/float) → используется как есть.
- Если строка: пробелы и неразрывные пробелы (`\xa0`) удаляются, запятая в десятичном разделителе заменяется на точку, затем выполняется `float(строка)`.
- Пустое или нечисловое → **0.0**.

Формально:

```
parse_quantity(value):
  если value — число → value
  иначе строка: убрать пробелы, заменить "," на ".", float(строка) или 0.0
```

### 3.2. Выбор остатка по материалу

Для каждого материала ищутся строки в Excel по алиасам (столбцы 1 и 2). По найденным строкам берётся значение из столбца «Сальдо на конец периода».

- При нескольких совпадениях приоритет: **точное совпадение** алиаса важнее частичного; среди одинаковых по точности выбирается строка с **максимальным** значением сальдо.
- Если среди выбранных все значения 0, подставляется значение из **первой** найденной строки (может быть 0).

То есть «лучший» остаток по материалу — это одно число (в кг), которое дальше подставляется в формулы раздела 1.

## 4. Итоговые зависимости

1. **Остатки** по материалам → из Excel (`extract_balances`), в кг.
2. **Рецепты** → из `config/recipes.yaml`: название бетона и расход каждого материала на 1 м³ (кг/м³).
3. **Максимум м³** по каждому типу бетона → формулы 1.1 и 1.2.
4. **Нужно материалов (кг)** → формула 1.3.
5. **Стоимость** → формула 2, цены за м³ из `config/prices.yaml`.
